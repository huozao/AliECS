# 贡献与提交流程（中文约定）

## 1. 分支与提交说明
- 分支命名建议使用中文语义拼音或英文短语，但**PR 标题与说明必须使用中文**。
- Commit message 建议中文，保持“做了什么 + 为什么做”。

示例：
- 分支：`feature/ecs-部署健壮性`
- 提交：`完善 ECS 部署脚本回滚日志`
- PR 标题：`完善 ECS 部署失败回滚与健康检查说明`

## 2. 提交流程
1. 先本地验证通过。
2. 推送分支并发起 PR（标题/描述中文）。
3. 合并前确认 CI 通过。

## 3. 变更边界提醒
- 不容器化 Nginx 和 sing-box。
- Actions 只做交付编排，不承担运行时职责。
- 部署顺序必须保持：先迁移后切换。
