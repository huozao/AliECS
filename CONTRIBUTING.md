# 贡献与提交流程（中文约定）

## 1. 提交与 PR 规则
- 所有 **commit message 必须使用中文**。
- 所有 **PR 标题必须使用中文**。
- 每次功能修改必须同步更新：
  - `VERSION`
  - `CHANGELOG.md`

推荐提交/PR格式：
- `v2.1.4：完善版本号自动发布流程`
- `v2.1.5：修复 Admin UI 登录状态显示`
- `v2.2.0：新增配方查询入口与库存查询导航`

## 2. 版本号递增规则
- patch：仅修 bug 或小调整（`v2.1.3 -> v2.1.4`）
- minor：新增功能但不破坏流程（`v2.1.4 -> v2.2.0`）
- major：破坏性大改或架构重构（`v2.2.0 -> v3.0.0`）

## 3. 发布触发规则
- push `v*` tag：按 tag 触发发布部署。
- workflow_dispatch：
  - 若填写 `image_tag`，优先用输入版本。
  - 若不填，自动读取 `VERSION`。

## 4. 变更边界提醒
- 不容器化 Nginx 和 sing-box。
- Actions 只做交付编排，不承担运行时职责。
- 部署顺序必须保持：先迁移后切换。
