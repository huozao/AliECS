自动部署流程概览

  1. 本地开发并通过 local/docker-compose.local.yml 验证
  2. push 到 GitHub，触发 CI 检查
  3. 创建版本标签（例：v0.1.2）或手工 workflow_dispatch
  4. Actions 构建并推送镜像：public-web / admin-ui / backend-api
  5. SSH 到 ECS 执行 /opt/app/deploy/ecs/deploy.sh <tag>
  6. 先迁移、再切换、再健康检查，失败自动回滚
