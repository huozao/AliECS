AliECS 面向 AI / 开发工具型项目：保留本地快速验证体验，同时将生产发布标准化为 GitHub tag → Actions → GHCR → ECS deploy.sh。

查看就绪状态 自动部署指南

项目简介

本地与 ECS 手工链路均可运行，当前重点是把手工流程收敛为稳定自动发布链路。

架构概览

宿主机 Nginx 统一入口；应用由 public-web/admin-ui/backend-api/postgres 组成并在 Docker Compose 运行。

自动部署能力

支持 tag 发布与 workflow_dispatch；deploy.sh 内置迁移、健康检查、失败回滚和日志落盘。

文档入口

仓库内提供自动部署总说明、首次部署清单和 ECS 运维记录模板，可直接复用到新服务器。

自动部署流程概览

  1. 本地改代码并通过 local/docker-compose.local.yml 验证
  2. 推送到 GitHub，并创建版本标签（如 v0.1.0）
  3. Actions 构建并推送三类镜像到 GHCR
  4. SSH 到 ECS 执行 /opt/app/deploy/ecs/deploy.sh <tag>
  5. 执行迁移、切换、健康检查；失败自动回滚
